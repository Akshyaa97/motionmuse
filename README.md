# MotionMuse

Motion that fits your product, not the trend. MotionMuse is a Figma plugin that helps designers find and apply the perfect animations for their UI elements based on industry standards and brand personality.

## 🚀 Features

- **Intelligent Recommendations:** Get animation suggestions tailored to your specific industry (Fintech, E-commerce, Gaming, etc.) and brand vibe (Playful, Trustworthy, Luxurious, etc.).
- **Smart Layer Detection:** Automatically detects the type of element you've selected (Button, Card, Modal, etc.) to provide relevant motion presets.
- **Live Interactive Previews:** See animations in action directly within the plugin UI before applying them to your designs.
- **One-Click Implementation:** Automatically wire up "Smart Animate" prototype connections in Figma with a single click.
- **Developer-Ready Code:** Copy production-ready CSS keyframes or Framer Motion code for every animation.
- **Customizable:** Overwrite auto-detected elements to explore different motion possibilities.

## 🛠 Installation (Local Development)

1. **Download the Plugin:**
   Click the green **<> Code** button at the top of this repository and select **Download ZIP**.
2. **Extract the Folder:**
   Unzip the downloaded file to a permanent folder on your computer.
3. **Open the Figma Desktop App:**
   Launch Figma and open any design file.
4. **Import the Plugin:**
   Go to the menu (Figma logo) > **Plugins** > **Development** > **Import plugin from manifest...**
5. **Select Manifest:**
   Locate the unzipped folder and select the `manifest.json` file.
6. **Run MotionMuse:**
   Right-click on your canvas > **Plugins** > **Development** > **MotionMuse**.

## 📖 How to Use

1. **Select an element** on your Figma canvas.
2. **Choose the industry** and **vibe** that matches your project.
3. **Browse the suggested animations** and watch the live previews.
4. **Click "Apply to selection"** to create a prototype flow automatically.
5. **Copy the code** to share with your developers.

## 💻 Development Workflow

This plugin uses a simple build system to bundle assets into a single `ui.html` file, as required by the Figma Plugin API.

### Project Structure
- `code.js`: The main plugin logic (runs in the Figma sandbox).
- `ui.html`: The user interface (generated by `build.py`).
- `ui.js`: Source JavaScript for the UI.
- `ui.css`: Source CSS for the UI.
- `data/animations.json`: The library of animation presets.
- `build.py`: Python script to bundle CSS, JS, and Data into `ui.html`.

### Building
After making changes to `ui.js`, `ui.css`, or `data/animations.json`, run the build script to update the plugin UI:

```bash
python3 build.py
```
